# UI/UX Checklist

Use this checklist before editing and again before wrapping up.

## Role-first review

- Who is using this page: admin, team lead, department head, director, or public reporter?
- What is the main task they came here to complete?
- What should they notice in the first 5 seconds?

## Layout review

- Is the page title clear?
- Is the main action obvious?
- Are summary, filters, content, and history separated clearly?
- Is there too much repeated chrome or too many equal-weight buttons?

## Table review

- Are important columns near the left?
- Are statuses visually scannable?
- Are row actions easy to find without clutter?
- Can the user understand the table without reading every column heading twice?

## Form review

- Are fields grouped by meaning?
- Are required fields easy to spot?
- Are helper texts short and useful?
- Is the submit action visible without hunting?

## Workflow review

- Can the user tell the current status of the report?
- Can the user see what to do next?
- Can the user understand who owns the case now?

## Visual review

- Keep Sarabun and the existing green brand tone.
- Prefer stronger spacing and hierarchy over extra colors.
- Use cards, dividers, badges, and section headers consistently.
- Avoid cramped sections and long walls of text.
